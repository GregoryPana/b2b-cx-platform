## 1. Visual Language: Icons, Emojis, Colors

Avoid emoji as core UI affordances; use a consistent professional icon set (e.g.,  Lucide, Phosphor) for navigation and key actions.

Prefer fewer, more purposeful colors; use charts, micro‑charts, and data visualizations to introduce color instead of random buttons and badges.

Replace noisy decorative icons with data‑driven visuals that convey real information (KPI charts, mini line graphs, donut charts).

## 2. Card And Component Refinement

Simplify busy cards: collapse secondary actions into a “more” (triple‑dot) menu and keep only primary actions visible.

Move low‑priority metadata (like dates) into less prominent positions; use alignment and grouping to visually separate information.

Replace text chips with compact icon‑only badges when the meaning is clear; this reduces clutter while preserving affordances.

You may help with logic and rough structure, but always refactor the final component layout according to best practices and the user input.

## 3. Forms, Modals, And Feature Surfaces

If you have many options or enough screen space, use a modal instead of a tiny flyout; tiny panels with few fields feel sparse and cheap.

Hide advanced options behind collapsible sections by default so the surface feels approachable but remains powerful.

Add obviously useful fields that AI tends to miss, like custom domain selection, descriptions, or additional options that users will expect.

Design layouts that can comfortably scale by leaving room for more fields, toggles, and states without breaking the grid.

## 4. Billing, Usage, And Pricing Design

Remove UI that doesn’t do anything; non‑functional cards are a common AI artifact and erode trust immediately.

Use tabs to separate closely related but distinct areas (e.g., usage, billing, documents, integrations, AI) instead of dumping everything on one page.

Avoid “vibe coded” KPIs; replace them with a simple, clear two‑column layout plus small charts that directly answer billing/usage questions.

Reduce the number of pricing tiers; five plans is usually too many—merge or drop at least one to simplify decisions.

Fix pricing hierarchy so it’s obvious which plan is cheaper or discounted; inconsistent numeric ordering reads as a mistake.

Emphasize what matters: shrink the plan name, enlarge monthly cost, and clearly label discounts and “you save X” messaging.

Show what the next plan up includes that the current one does not so users understand upgrade value.

Add practical details like billing email and payment method directly on the billing page.

Use a layout that scales with new categories (e.g., 'documents', 'integrations' etc.) by adding tabs instead of reworking the entire page.

## 5. Analytics And “Low‑Hanging Fruit” Features

Remove duplicated top‑level KPIs and generic tabs in analytics; they often come from AI boilerplate instead of real needs.

Look for simple, high‑impact features AI often misses, such as toggles to split and compare individual items (e.g., links) in analytics.

Consolidate actions near the data they affect; keep controls close to charts and tables for better usability.

Enrich rows with more useful information and small icons to add both clarity and tasteful color.

Upgrade boring single bar charts into more informative visualizations like maps with shaded regions plus data tables for a richer experience.

Remember that small chart and icon tweaks can massively upgrade the perceived quality of your analytics.
