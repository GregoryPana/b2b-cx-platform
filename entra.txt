  "name": "John Smith",

--------------------------------
AUTHENTICATION FLOW
--------------------------------

1. User opens the React application.
2. React authenticates the user using Microsoft Entra via MSAL.
3. User signs in with their organizational account.
4. Entra returns tokens to the frontend.
5. React obtains an access token for the backend API.
6. React includes the token in API requests:

Authorization: Bearer <access_token>

7. FastAPI validates the JWT token.
8. FastAPI extracts the claims.
9. FastAPI authorizes access based on roles.

Each module has its own role permissions.
